# QuantFlow 平台验证工作流

## 推荐验证口径

A1 龙虎榜资金追踪因子推荐按“动态龙虎榜事件池”验证。

本因子回答的问题是：

> 当天已经上过龙虎榜的股票里，哪些股票因为买入席位更强，次日开盘更容易有溢价。

因此，首选股票池为：

> 每个交易日当天上过龙虎榜的股票。

这不是固定股票名单，而是每天变化的事件池。

## 不建议混用的口径

不要在同一次验证中同时混用：

- 沪深全A；
- 中证全指；
- 平台默认全市场股票池；
- 动态龙虎榜事件池。

全市场验证和事件池验证回答的是不同问题。全市场结果接近 0 或为负，不能直接否定事件池内排序能力。

## 事件池验证设置

### 股票池

每个交易日 `T` 只保留当天出现在龙虎榜列表中的股票。

要求：

1. 股票 `S` 在交易日 `T` 上龙虎榜，才进入 `T` 日样本；
2. 股票 `S` 只是在其他日期上过龙虎榜，不能进入 `T` 日样本；
3. 不能把回测期内所有上过龙虎榜的股票做成静态名单；
4. IC、Rank IC、分组收益和回测曲线都必须只在当天事件池内部计算。

### 数据接口

使用 pandadata 官方接口：

- `get_lhb_list`
- `get_lhb_detail`
- `get_stock_daily`
- `get_trade_cal`

### 必要字段

龙虎榜列表：

- `trade_date`
- `ts_code`
- `lhb_type` 或 `lhb_reason`

龙虎榜明细：

- `trade_date`
- `ts_code`
- `agency_name`
- `buy_value`
- `sell_value`

日行情：

- `trade_date`
- `ts_code`
- `open`
- `close`

## 数据清洗

1. 统一股票代码格式；
2. 统一日期格式；
3. 删除席位名称为空的记录；
4. 买入金额和卖出金额转为数值；
5. 同一交易日、同一股票、同一席位出现多行时，先合并再计算；
6. 同一交易日、同一股票最终只保留一条因子记录。

## 收益标签

信号产生时间为交易日 `T` 收盘后。

回测标签为：

```text
next_open_premium = 下一交易日开盘价 / T 日收盘价 - 1
```

标签只能用于回测，不能参与 `T` 日因子计算。

## 防未来函数

计算交易日 `T` 的因子时，只能使用 `T` 日之前的席位历史。

要求：

1. 席位历史样本满足 `history_trade_date < T`；
2. 不使用 `T` 日之后的龙虎榜数据；
3. 不使用 `T` 日之后的行情或收益；
4. 生产因子文件不包含 `next_open_premium`、`next_open`、`future_return` 等字段。

## 因子计算

席位历史窗口为过去 30 个交易日。

席位历史收益：

```text
历史样本次日开盘溢价 = 历史样本下一交易日开盘价 / 历史样本龙虎榜日收盘价 - 1
```

席位统计指标：

- 胜率；
- 盈亏比；
- 平均次日开盘溢价；
- 有效样本数。

席位得分：

```text
seat_score =
100 * (
  0.40 * win_rate
  + 0.25 * payoff_component
  + 0.25 * premium_component
  + 0.10 * sample_component
)
```

其中：

```text
payoff_component = min(max(payoff_ratio, 0), 3) / 3
premium_component = min(max((mean_premium + 0.03) / 0.08, 0), 1)
sample_component = min(sample_count / 10, 1)
```

股票因子值：

```text
raw_seat_score = sum(seat_score * buy_value) / sum(buy_value)
factor_value = raw_seat_score * (0.75 + 0.25 * amount_rank)
score = factor_value 在当日事件池内做分位排名后乘以 100
```

其中 `amount_rank` 为该股票当日买入总金额在当日事件池内的分位数。

## 因子列选择

优先使用：

```text
factor_value
```

也可以使用：

```text
score
```

不要使用：

```text
rank
```

`rank` 是名次，数值越小代表越好。若把 `rank` 当作普通因子值，会把方向算反。

## 建议输出指标

- 样本数；
- 交易日数；
- 每日样本数；
- IC；
- Rank IC；
- IC 标准差；
- ICIR；
- p-value；
- 五分组收益；
- 多空收益；
- 胜率；
- 夏普；
- 最大回撤；
- 换手率；
- 月度表现；
- 按 `lhb_type` 或 `lhb_reason` 分层后的表现。

## 结果核对

平台结果需要重点核对：

1. 每日样本数是否大约为几十只，而不是几百或几千只；
2. 股票池是否为每日动态事件池；
3. 因子列是否为 `factor_value` 或 `score`；
4. 是否误用了 `rank`；
5. 是否存在同一股票、同一日期、同一席位重复计入；
6. 生产因子文件是否不包含未来收益字段。

## 全市场验证

如需验证它是否适合作为全市场通用因子，应单独开一组验证，不与事件池验证混合。

全市场口径下：

1. 股票池使用沪深全A；
2. 当天上过龙虎榜且能计算 A1 的股票使用其因子值；
3. 当天没有上龙虎榜的股票，因子值填 0 或缺失，并在报告中说明；
4. IC 和分组收益在全市场股票池内部计算。

全市场结果若接近 0 或为负，只说明该因子不适合作为普通全市场日频 Alpha，不能直接否定其事件池内排序用途。
